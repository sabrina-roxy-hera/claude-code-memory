# /commit — 存档当前对话

## 规则

根据用户参数决定操作：
- `/commit` 或 `/commit` 后跟非 end 内容：**存档**（阶段性保存）
- `/commit end`：**结档**（任务完成）

## 存档（commit）

1. 读取当前工作目录，确定项目名
2. 整理当前 session 对话中的：进度、结论、决策、待办、续做入口
3. 写入存档文件到当前工作目录：`<任务名>_存档.md`
4. 在 `.session.log` 追加一行：`[YYYY-MM-DD HH:MM] DONE 描述 → 产出物路径`

## 结档（commit end）

1. 执行上述存档步骤
2. 额外更新 `~/.claude/CLAUDE.md` 的 Session Log 和 Short-term Memory
3. 回顾本次 session 中用户的隐含偏好和习惯，更新到 CLAUDE.md 的 Long-term Preferences

### 偏好识别规则

回顾对话时，识别并记录三类偏好：

**第一类：用户明确说出的规则**
- 用户直接下达的指令或约束
- 例："飞书文档里的图片只用 docx 嵌入，不要用 Block API"
- 写入方式：原文记录，标注来源对话

**第二类：用户在修改中反复表现出的偏好**
- 用户没有明说，但通过多次修改、删除、纠正表现出的倾向
- 例：用户每次都删掉 AI 写的 emoji → 归纳为"不喜欢 emoji"
- 写入方式：归纳总结，标注"行为归纳"

**第三类：只适用于特定场景的要求**
- 不是全局生效的规则，而是特定任务类型下的约束
- 例："竞品调研必须每家都配截图" → 仅竞品调研场景生效
- 写入方式：记录规则 + 标注适用场景

## 存档文件格式

```markdown
# <任务名> 存档

## 状态：<阶段性完成 | 已结档>

## 已完成
- ...

## 待办
- ...

## 续做入口
- ...
```

# /commit — 存档当前对话 & 偏好记忆

## 路由规则

根据用户参数决定操作：
- `/commit` 或 `/commit` 后跟非 end 内容 → **存档**（阶段性保存）
- `/commit end` → **结档**（任务完成 + 偏好写入）

---

## 存档（/commit）

### 第一级：写入存档文件

1. 读取当前工作目录（`pwd`），确定项目名和任务名
2. 回顾当前 session 对话，整理：
   - **已完成**：本次做了什么，产出了什么（具体文件路径、链接）
   - **关键决策**：为什么选了 A 方案而不是 B（下次续做时需要知道的判断依据）
   - **待办**：还有什么没做完
   - **续做入口**：下次从哪里接手（文件路径、链接、命令）
3. 写入存档文件：`<当前工作目录>/<任务名>_存档.md`

### 第二级：追加 Session Log

在**当前工作目录**的 `.session.log` 追加一行：

```
[YYYY-MM-DD HH:MM] DONE <一句话描述> → <产出物路径>
```

文件不存在时自动创建。

---

## 结档（/commit end）

先执行上述**完整存档步骤**，然后额外执行偏好识别和记忆写入。

### 第三级：偏好识别 → 写入 CLAUDE.md

回顾本次 session 的**完整对话历史**，识别以下三类偏好：

#### 第一类：用户明确说出的规则

用户直接下达的指令、约束、禁令。

识别方式：用户说了"不要…"、"必须…"、"以后都…"、"别再…"等明确指令。

示例：
- "图片只用 docx 嵌入，不要用 Block API"
- "文章不要用 emoji"
- "写完文档必须通知我"

写入格式：直接记录原文，标注来源场景。

#### 第二类：用户在修改中反复表现出的偏好

用户没有明说，但通过**多次修改、删除、纠正、重写**表现出的倾向。这类偏好最有价值——用户自己可能都没意识到。

识别方式：
- 用户删掉了 AI 写的某类内容（连续 2 次以上）→ 归纳为"不喜欢这类内容"
- 用户每次都把 AI 的输出改成某种风格 → 归纳为"偏好这种风格"
- 用户反复追问某类细节 → 归纳为"重视这类信息"

示例：
- 用户每次都删掉文章里的 emoji → "写作风格：不使用 emoji"
- 用户每次都把表格改成描述段落 → "排版偏好：避免表格，用文字描述"
- 用户总是要求加上真实案例 → "内容偏好：案例要真实具体，不要虚构"

写入格式：归纳总结 + 标注"行为归纳"。

#### 第三类：只适用于特定场景的要求

不是全局规则，而是只在特定任务类型下才生效的约束。**必须标注适用场景**，避免在不相关的任务里过度套用。

示例：
- "竞品调研必须每家都配截图" → 适用场景：竞品调研
- "标题用「开源X｜价值点」格式" → 适用场景：公众号写作

写入格式：规则 + `[场景：xxx]` 标签。

### 写入位置

目标文件：`~/.claude/CLAUDE.md`

**如果文件不存在**，先从模板创建：
```bash
cp <commit.md 同目录>/CLAUDE.md.template ~/.claude/CLAUDE.md
# 或手动创建含以下 section 的文件：
# ## Long-term Preferences
# ## Short-term Memory
# ## Session Log
```

写入三个区域：

1. **Session Log**：在 `## Session Log` 区追加本次任务记录（一行）
2. **Short-term Memory**：更新 `## Short-term Memory` 区（当前阶段需要跨 session 记住的临时信息，如项目路径、当前阶段、待确认事项）
3. **Long-term Preferences**：将识别到的三类偏好追加到 `## Long-term Preferences` 区
   - 不要删除已有条目
   - 如果新偏好与已有条目冲突，以新的为准，更新旧条目
   - 相同偏好不重复写入

### 写入原则

- **宁少勿滥**：只写真正有信号强度的偏好，不要把一次性的随口一说当成长期偏好
- **可执行**：每条偏好都应该是 AI 下次能直接遵守的指令，不要写模糊的感受
- **带上下文**：让未来的 AI 能理解这条偏好是怎么来的
- **Short-term Memory 不放敏感信息**：API Token、密码等凭证不写入，存到安全的地方

## 存档文件格式

```markdown
# <任务名> 存档

## 状态：<阶段性完成 | 已结档>

## 已完成
- 做了什么，产出了什么（附路径/链接）

## 关键决策
- 选了方案 A 而不是 B，原因是…（下次续做需要知道）

## 待办
- 还剩什么没做

## 续做入口
- 文件：`路径/文件名`
- 命令：`cd 项目目录 && 执行命令`
- 链接：https://...
```

---

## 使用示例

**阶段性保存**（还没做完，要去吃饭）：
```
/commit
```

**任务完成，写入偏好记忆**：
```
/commit end
```

**带备注存档**：
```
/commit 完成了登录模块，下次继续做注册
```

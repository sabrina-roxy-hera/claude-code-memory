# /reset — 存档并开启新 session

上下文快满了？任务告一段落？用 `/reset` 存档当前进度，然后开新会话继续。

---

## 执行步骤

### 第一步：存档当前对话

1. 读取当前工作目录，确定项目名和任务名
2. 回顾当前 session 对话，整理：
   - **已完成**：本次做了什么，产出了什么（具体文件/链接）
   - **关键决策**：选了哪个方案，为什么（续做时需要知道）
   - **待办**：还有什么没做完
   - **续做入口**：下次从哪个文件/命令/链接接手
3. 写入存档文件：`<当前工作目录>/<任务名>_存档.md`
4. 在当前工作目录的 `.session.log` 追加一行记录：
   ```
   [YYYY-MM-DD HH:MM] DONE <一句话描述> → <存档文件路径>
   ```

### 第二步：告知用户

存档完成后，输出：

```
存档已写入：<存档文件路径>

下次续做时，把存档文件发给我，或者说"继续 <任务名>"，我会先读存档再接着干。

现在可以用 Ctrl+R（新建会话）或直接关闭当前对话窗口开启新 session。
```

---

## 存档文件格式

```markdown
# <任务名> 存档

## 状态：阶段性完成

## 已完成
- 做了什么，产出了什么（附路径/链接）

## 关键决策
- 选了方案 A 而不是 B，原因是…

## 待办
- 还剩什么没做

## 续做入口
- 文件：`路径/文件名`
- 命令：`cd 项目目录 && 执行命令`
- 链接：https://...
```

---

## 下次续做

新 session 开始时，直接说：

```
继续上次的 <任务名>
```

或把存档文件路径发给 AI，它会自动读取进度再接手。

---

## /reset vs /commit 的区别

| | /commit | /commit end | /reset |
|---|---|---|---|
| 写存档文件 | ✅ | ✅ | ✅ |
| 更新 Session Log | ✅ | ✅ | ✅ |
| 写入 CLAUDE.md 偏好 | ❌ | ✅ | ❌ |
| 提示开新 session | ❌ | ❌ | ✅ |
| **适合场景** | 中途保存 | 任务完成 | 上下文快满 |
